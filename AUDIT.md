# moldcell.md website review

Reviewed scope:
- Main moldcell.md website
- Desktop layout
- Mobile responsiveness
- Main navigation and footer

## Finding 1

**Area:** UI

**Observation:**  
On desktop, the main content is constrained to a relatively narrow container of around 960 px. On large screens this leaves a lot of unused horizontal space and makes sections such as banners, cards and content blocks feel smaller than expected.

**Why it matters:**  
A narrow desktop layout can make the page feel outdated and less intentional on modern laptop and desktop screens. It also limits how much useful content can be shown without extra vertical scrolling.

**Suggested fix:**  
Increase the desktop container width to a more modern range, for example 1280-1400 px. Keep text-heavy blocks narrower for readability, but allow visual sections, cards and promotional blocks to use more available screen width.

## Finding 2

**Area:** Responsive design

**Observation:**  
The mobile layout does not adapt cleanly on small screens. Some sections appear to keep desktop-like spacing, sizing and structure instead of being redesigned for a 360 px viewport.

**Why it matters:**  
Many telecom users will visit the site from mobile devices. If the layout is cramped, hard to scan or requires unnecessary scrolling, users may abandon common tasks such as checking offers, finding stores or topping up.

**Suggested fix:**  
Review the main pages at 360 px, 390 px and tablet widths. Use mobile-first section layouts, reduce horizontal spacing, stack content intentionally, and make primary actions easy to reach on small screens.

## Finding 3

**Area:** UX

**Observation:**  
The main navigation and footer expose a large number of links and categories at once. The footer contains several columns of links plus a separate row of shortcut actions, which makes the information hierarchy feel dense.

**Why it matters:**  
Users usually arrive with a small number of high-intent tasks: recharge, choose a plan, switch to Moldcell, find a store or contact support. When too many links have similar visual weight, the most important paths become harder to identify.

**Suggested fix:**  
Prioritize the top user journeys and group secondary links more clearly. In the footer, separate high-priority actions from legal/company/help links and reduce the visual competition between shortcut buttons and long link lists.
